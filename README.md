# PshchEdge

Welcome to the PshchEdge project. This application provides a comprehensive trading setup encompassing behavioral tracking, trade journaling, AI signals, and market alerts.

## Installation Procedures

1. **Clone the repository** (if you haven't already).
2. **Install dependencies**:
   ```bash
   npm install
   ```
3. **Set up the database**:
   Configure your environment variables in `.env` (e.g., `DATABASE_URL`), then push the schema:
   ```bash
   npm run db:push
   ```
4. **Run the development server**:
   ```bash
   npm run dev
   ```
5. **Build for production**:
   ```bash
   npm run build
   npm start
   ```

## API Documentation

The server exposes the following REST APIs under the `/api` prefix.

### Trades
- **`GET /api/trades`**  
  Retrieves all recorded trades.
- **`POST /api/trades`**  
  Creates a new trade entry. If specific emotional states (like "revenge" or "excited") are provided, behavioral logs are automatically generated.
- **`PATCH /api/trades/:id`**  
  Closes an active trade by providing an exit price and optional discipline score.

### Watchlist & Signals
- **`GET /api/watchlist`**  
  Retrieves current items on the watchlist.
- **`GET /api/signals`**  
  Retrieves AI signals generated for trading.
- **`POST /api/refresh/market`**  
  Refreshes background market data.
- **`POST /api/refresh/signals`**  
  Regenerates AI trading signals.

### Alerts
- **`GET /api/alerts`**  
  Retrieves all configured alerts.
- **`POST /api/alerts`**  
  Creates a new price, volume, or technical indicator alert.
- **`PATCH /api/alerts/:id`**  
  Updates the status (e.g., toggles `isActive`) of a specific alert.
- **`DELETE /api/alerts/:id`**  
  Deletes a specific alert.

### Behavioral Logs
- **`GET /api/behavioral-logs`**  
  Retrieves all behavioral and discipline logs generated by user actions or automatically parsed from low-discipline trade exits.
